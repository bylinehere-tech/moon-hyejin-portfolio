# 문혜진 감독 포트폴리오 마이크로사이트 — 에이전트 설계서

> **문서 버전**: v1.0  
> **작성일**: 2026-04-15  
> **목적**: Claude Code가 마이크로사이트를 자율적으로 구현하기 위한 통합 계획서  
> **실행 환경**: VS Code + Claude Code (Windows)

---

## 1. 작업 컨텍스트

### 1.1 배경

문혜진 감독은 공연예술 분야 10년+ 경력의 영상 전문가다. 홍보영상 제작과 무대영상 디자인이라는 두 축을 동시에 수행하며, 80편 이상의 작품에 참여했다. 현재 포트폴리오를 체계적으로 전시하고 신규 클라이언트의 프로젝트 문의를 접수할 전용 마이크로사이트가 필요하다.

### 1.2 목적

1. 감독의 전문성과 작품 세계를 시각적으로 증명하는 **포트폴리오 허브** 구축
2. 공연 제작사·극장·아티스트가 **프로젝트 문의를 간편하게** 할 수 있는 전환 경로 설계
3. 홍보영상과 무대영상을 **명확히 구분**하여 각각의 전문성이 돋보이도록 설계

### 1.3 타겟 오디언스

| 순위 | 대상 | 행동 패턴 |
|------|------|----------|
| 1차 | 공연 제작사 (쇼노트, HJ컬쳐, 더블케이 등) | 신작 공연의 영상 파트너 탐색 → 포트폴리오 확인 → 문의 |
| 2차 | 극장·문화재단 (정동극장, CJ문화재단 등) | 자체 기획 공연 영상 외주 의뢰 → 이력 확인 → 문의 |
| 3차 | 아티스트·개인 기획사 | 콘서트·리사이틀 무대영상 필요 → 사례 확인 → 문의 |

### 1.4 핵심 포지셔닝

> **"무대 위의 이야기를, 영상으로 완성합니다"**

차별점: 공연예술 도메인 안에서 홍보영상(마케팅)과 무대영상 디자인(연출)을 동시에 수행한다는 이중 역할.

### 1.5 범위 정의

| 포함 | 제외 |
|------|------|
| 프론트엔드 빌드 (React + Vite) | 백엔드 서버 구축 |
| 플레이스홀더 기반 스캐폴딩 | 실제 영상·이미지 에셋 편집 |
| 포트폴리오 데이터 구조 (JSON) | CMS 연동 (향후 고려) |
| 문의 폼 연동 (Web3Forms) | 도메인 구매·DNS 설정 |
| Vercel 배포 설정 | SEO 고도화, 애널리틱스 심화 |

### 1.6 제약조건

- **콘텐츠 업데이트 주체**: Jay (개발자). 감독은 코드 수정 불가
- **데이터 관리 방식**: JSON 파일 수정 → 재배포 (Vercel auto-deploy via Git push)
- **현재 콘텐츠 상태**: 미확보. 플레이스홀더로 영역만 구성하고 이후 교체
- **사이트 언어**: 한국어 전용
- **디자인 톤**: 다크 시네마틱 (블랙/차콜 배경 + 화이트/골드 타이포)

### 1.7 용어 정의

| 용어 | 의미 |
|------|------|
| 홍보영상 | 공연의 티저·트레일러·프로모션 영상. 독립된 영상물로 YouTube에 존재 |
| 무대영상 (프로젝션 디자인) | 공연 무대 위 프로젝션·백그라운드·인터랙티브 효과. 공연 맥락 속에서만 존재 |
| 쇼릴 | 대표작 하이라이트를 60~90초로 교차 편집한 릴 영상 |
| 스캐폴딩 | 실제 콘텐츠 없이 레이아웃·컴포넌트·데이터 구조만 완성한 뼈대 |

## 2. 기술 스택

| 영역 | 선택 | 근거 |
|------|------|------|
| 프레임워크 | React 18 + Vite | Jay의 기존 작업 스택. 빠른 빌드, HMR 지원 |
| 스타일링 | Tailwind CSS 3 | 다크 테마 유틸리티, 반응형 빠른 구현 |
| 애니메이션 | Framer Motion | 스크롤 트리거 애니메이션, 섹션 전환 |
| 폼 백엔드 | Web3Forms (무료 티어) | 서버리스, 이메일 알림, 스팸 방지 내장 |
| 영상 임베드 | YouTube (포트폴리오) + `<video>` 태그 (히어로) | 기존 YouTube 활용 + 히어로 자동재생 요건 |
| 배포 | Vercel | Git push 자동 배포, Vite 네이티브 지원, 무료 SSL/CDN |
| 아이콘 | Lucide React | 경량, Tailwind 호환 |
| 폰트 | Pretendard (본문) + 볼드 디스플레이 서체 (타이틀) | 한국어 최적화, 시네마틱 임팩트 |

## 3. 사이트 구조 & 섹션 정의

싱글 페이지 스크롤 구조. 방문자 여정: **임팩트 → 신뢰 구축 → 전환**

### 3.1 섹션 맵

| # | 섹션 ID | 섹션명 | 퍼널 단계 | 핵심 요소 |
|---|---------|--------|----------|----------|
| 01 | `hero` | 히어로 | 첫인상 | 쇼릴 자동재생(음소거·루프) + 감독명 + 포지셔닝 카피 + CTA 2개 |
| 02 | `promo-video` | 포트폴리오: 홍보영상 | 작품 전시 | 장르 필터 + 썸네일 그리드 + YouTube 라이트박스 + 작품 정보 |
| 03 | `stage-video` | 포트폴리오: 무대영상 | 작품 전시 | 장르 필터 + 카드 레이아웃 + 스틸컷 + 역할 태그 |
| 04 | `about` | 감독 소개 | 신뢰 구축 | 프로필 사진 + 연출 철학 + 숫자 카드 (경력·작품수·협업사수) |
| 05 | `clients` | 클라이언트 & 실적 | 신뢰 강화 | 로고 월 + 실적 카운터 + 타임라인(선택) |
| 06 | `contact` | 문의 | 전환 | 프로젝트 유형 드롭다운 + 간편 폼 5필드 + CTA |

### 3.2 내비게이션

- 상단 고정(sticky) 내비게이션 바
- 메뉴: 작품 (홍보영상 / 무대영상 서브메뉴) │ 감독 소개 │ 실적 │ 문의
- 모바일: 햄버거 메뉴 + 하단 고정 "프로젝트 문의" 플로팅 버튼
- 우측에 항상 "프로젝트 문의" CTA 버튼 (액센트 컬러 배경)

### 3.3 디자인 토큰

```
colors:
  bg-primary: #0A0A0A        # 메인 배경 (거의 블랙)
  bg-secondary: #1A1A1A      # 카드/섹션 배경
  bg-tertiary: #252525        # 호버, 서브 영역
  text-primary: #FFFFFF       # 메인 텍스트
  text-secondary: #A0A0A0     # 보조 텍스트
  accent: #C5A55A             # 골드 액센트 (CTA, 강조)
  accent-hover: #D4B96A       # 골드 호버
  border: #333333             # 경계선

typography:
  display: 볼드 디스플레이 서체 (구현 시 확정)
  body: Pretendard
  size-hero-title: 4rem ~ 6rem (반응형)
  size-section-title: 2rem ~ 3rem
  size-body: 1rem

spacing:
  section-padding: 80px ~ 120px (데스크톱) / 48px ~ 64px (모바일)
  
breakpoints:
  mobile: 0 ~ 767px
  tablet: 768px ~ 1023px
  desktop: 1024px+
```

### 3.4 반드시 피해야 할 것

- 밝은 배경 + 작은 썸네일 빽빽한 레이아웃
- 과도한 패럴랙스 / 트랜지션 (영상 콘텐츠 집중 방해)
- 장황한 텍스트 블록 (영상·이미지가 주인공)
- 홍보영상과 무대영상을 같은 갤러리에 혼합 (전문성 희석)

#### Step 4: 섹션별 컴포넌트 빌드

| 항목 | 내용 |
|------|------|
| **처리 주체** | 스크립트 + 에이전트 (레이아웃 판단, 카피 톤 결정) |
| **작업** | 6개 섹션 컴포넌트 각각 빌드 |
| **성공 기준** | 각 섹션이 플레이스홀더 데이터로 의도한 레이아웃을 정확히 표현 |
| **검증 방법** | LLM 자기 검증 (디자인 의도 대비 렌더링 결과 비교) + 규칙 기반 (빌드 에러 없음) |
| **실패 시** | 자동 재시도 (최대 2회). 레이아웃 의도 불일치 → 에이전트가 CSS 수정 판단 |

**섹션별 빌드 사양**:

**01. HeroSection**
- 풀스크린 뷰포트 (100vh)
- `<video>` 태그: 자동재생, 루프, 음소거, 플레이인라인 속성. 소스 경로는 `about.json`에서 참조 (현재 빈 값 → 플레이스홀더 다크 그라데이션 배경 표시)
- 모바일: 첫 프레임 스틸 대체 (poster 속성) → 탭 시 재생
- 텍스트 오버레이: 감독명 (대형 볼드) + 포지셔닝 카피 + CTA 2개 ("작품 보기" → #promo-video 스크롤, "프로젝트 문의" → #contact 스크롤)
- Framer Motion: 텍스트 페이드인 + 슬라이드업

**02. PromoVideoSection**
- `GenreFilter` 상단 배치
- 썸네일 그리드: 데스크톱 3열, 태블릿 2열, 모바일 1열
- 각 카드: 썸네일 + 호버 시 작품명·연도 오버레이
- 클릭 → `YouTubeLightbox` 열림
- 카드 하단: 작품명 / 장르 태그 / 연도 / 제작사
- `featured: true`인 작품만 기본 표시. "더 보기" 토글로 전체 리스트 확장
- Framer Motion: 카드 stagger 진입 애니메이션

**03. StageVideoSection**
- `GenreFilter` 상단 배치 (PromoVideoSection과 동일 UI)
- 카드형 레이아웃: 대표 스틸컷 + 작품명 + 장르 태그 + 역할 태그
- 클릭 시 확장 뷰: 스틸컷 2~3장 슬라이더 + 실황 클립 (있을 경우) + 역할 설명
- 역할 태그: "프로젝션 디자인" / "백그라운드 영상" / "인터랙티브 효과" 등 시각적 구분
- `featured: true`인 작품만 기본 표시

**04. AboutSection**
- 2컬럼 레이아웃: 좌측 프로필 사진 (플레이스홀더) / 우측 소개 텍스트
- 숫자 카드 3개: 경력 10년+ / 작품 80+ / 협업사 15+ (카운트업 애니메이션, 스크롤 트리거)
- 해외 이력 별도 라인: 도쿄·오사카·교토·북경·치앙마이
- 이중 역할 강조: 홍보영상 ↔ 무대영상 시각적 구분 (아이콘 또는 다이어그램)

**05. ClientsSection**
- 로고 월: 16개 협업사 로고 그리드 (플레이스홀더 박스 + 사명 텍스트)
- 실적 카운터: "80+ 작품" / "10년+ 경력" / "15+ 제작사"
- 타임라인(선택): 2015~2025 가로 스크롤. 구현 복잡도에 따라 에이전트가 포함 여부 판단

**06. ContactSection**
- 폼 5필드: 이름 / 소속 / 이메일 / 프로젝트 유형 (드롭다운: 홍보영상 의뢰·무대영상 디자인·기타 협업) / 메시지
- CTA: "프로젝트 상담하기"
- 대체 연락처: 이메일 + 카카오톡 링크 (선택)
- 제출 성공/실패 피드백 UI

### 6.5 데이터 전달 패턴

모든 데이터는 **파일 기반**으로 전달한다.

## 7. 콘텐츠 교체 운영 가이드 (README에 포함할 내용)

이 사이트는 감독이 직접 코드를 수정하지 않는다. 콘텐츠 업데이트 시 Jay가 아래 절차를 따른다.

### 7.1 작품 추가

1. `/src/data/promo-videos.json` 또는 `stage-videos.json`에 새 엔트리 추가
2. `featured: true`로 표시하면 메인 갤러리에 노출
3. 썸네일 이미지를 `/public/images/`에 추가하고 JSON의 `thumbnail` 필드에 경로 입력
4. YouTube 영상의 경우 `youtubeId` 필드에 영상 ID 입력 (URL에서 `v=` 뒤의 값)
5. Git commit & push → Vercel 자동 배포

### 7.2 클라이언트 로고 추가

1. SVG 또는 PNG 파일을 `/public/images/logos/`에 추가
2. `clients.json`에 새 엔트리 추가
3. Git push

### 7.3 감독 프로필 수정

1. `about.json`의 해당 필드 수정
2. 프로필 사진 교체 시 `/public/images/`에 새 파일 추가 후 경로 수정
3. Git push
